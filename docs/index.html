§01 — What it is

Read-only by design

Every skill calls the FREE tier of the jgs-magic-sysmlv1-mcp bridge. No writes, ever — your model is never modified.

One dispatcher

/jgs-v1 routes any request to the right specialist and flags PRO-only capabilities (writes, migration, cross-model).

Air-gapped

The skills talk only to your local bridge. SysML stays inside CATIA Magic — no cloud, no data export.

§02 — The skill set

/jgs-v1

Dispatcher — routes to the right specialist.

/jgs-v1-navigate

Structured model overview — packages, counts, diagrams.

/jgs-v1-inspect

Deep-dive one element — type, ports, relationships, links.

/jgs-v1-search

Full-text search with ranked, typed hits.

/jgs-v1-impact

Blast-radius analysis before changing a shared element.

/jgs-v1-audit

Six-specialist health audit with severity-graded findings.

/jgs-v1-report

Shareable model-health report + Requirements Traceability Matrix.

/jgs-v1-diagrams

Inventory and export diagrams; compare layout styles.

/jgs-v1-units

Look up units, quantity kinds, and library types.

/jgs-v1-fixplan

Turn audit findings into a read-only remediation plan.

…plus the audit specialists (naming, docs, requirements, duplicates, unused, methodology) and /jgs-v1-status. 17 skills in total — see SKILLS.md.

§05 — Install

Prerequisite: the jgs-magic-sysmlv1-mcp bridge installed and running (FREE tier), with a SysML v1 project open in CATIA Magic.

# clone, then install for Claude Code (default)
python install.py

# preview without writing anything
python install.py --dry-run

# install for another agent (or all user-global agents)
python install.py --agent gemini
python install.py --agent all

Skills install flat for short-name discovery (/jgs-v1). Restart your agent afterwards.
